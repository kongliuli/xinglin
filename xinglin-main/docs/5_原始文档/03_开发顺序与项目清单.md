# 开发顺序与项目清单

开发按以下顺序推进，确保早期可演示、后期可扩展：

## 第一阶段：基础类库（1–2 周）
- 开发通用模型与接口库（定义病人、报告、模板等对象）。
- 开发数据访问与文件工具库（用于读写数据库和本地文件）。

## 第二阶段：WPF 客户端核心功能（2–3 周）
- 实现报告录入界面，支持动态加载 JSON 模板并绑定数据。
- 实现 A4 纸精准打印与预览功能。

## 第三阶段：数据源适配器（1–2 周）
- 开发插件化适配器框架，支持动态加载不同数据源模块。
- 实现 HIS 数据库查询、身份证读卡器读取、第三方 API 调用三种适配器。

## 第四阶段：服务端开发（2 周）
- **内网服务**：提供报告归档接口和安全查询接口。
- **外网服务**：对接微信 OAuth，转发查询请求，生成 HTML 报告。

## 第五阶段：授权与管控（1 周）
- **主控服务**：实现机器码注册、心跳校验、模板下发、权限分发。

## 最终形成 6 个主要项目
1. **Core**（通用模型）
2. **Infrastructure**（数据/文件工具）
3. **Client.WPF**（主程序）
4. **Printing.Service**（打印引擎）
5. **DataAdapters**（适配器框架）
6. **Server.API**（内网+外网+授权服务）
